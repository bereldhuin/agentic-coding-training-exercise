## Exercise: Implement the Price Filter Feature using **planning**

---

### Goal

Practice the **plan-then-act** workflow by implementing a new feature in the Lebonpoint platform. You will ensure that the implementation is consistent with the existing architecture and API contract by first creating and reviewing a detailed plan.

By the end, you will have implemented the "search by price" feature on your chosen backend(s) using a structured approach.

---

#### Requirements:

- **Create a Plan First**: Start by asking your AI assistant (Claude, Codex, etc.) to "Create a plan to implement the price filter feature in the [chosen language] backend".
- **Review and Refine**: Carefully review the generated plan. Ensure it covers all the elements needed in a traditional software development workflow. _(The assignment is deliberately open-ended to encourage critical thinking about the steps involved.)_
- **Implementation**: Once the plan is solid, authorize the execution of the changes.
- **Verification**: Verify that the feature works as expected and doesn't break existing functionality.

---

### Hints

- Refer to the [`openapi.yaml`](/server/api/openapi.yaml) file for the exact schema and endpoint details.
- Use the techniques discussed in class to guide your AI assistant in generating a comprehensive plan.
